# JSON Visualizer & Analyzer - Product Specification

## 1. プロダクト概要 (Product Overview)
JSONデータをブラウザ上で簡単に可視化・分析できるフロントエンド完結型のウェブアプリケーション。
開発者やデータアナリストが、APIレスポンスやログデータなどの構造化データを素早く把握し、データ品質（Null率、Enum値の分布など）を評価することを目的とする。

## 2. メタデータ (Metadata)
- **プロダクト名**: JSON Visualizer & Analyzer
- **バージョン**: 1.0.0
- **アーキテクチャ**: フロントエンド完結型 (SPAライク)
- **技術スタック**: 
  - HTML5
  - CSS3 (Vanilla CSS, CSS Variables, Flexbox/Grid)
  - JavaScript (Vanilla JS, ES6+)
- **依存関係**: 外部ライブラリなし (Zero dependencies)
- **フォント**: Inter, JetBrains Mono (Google Fonts)

## 3. 機能要件 (Functional Requirements)

### 3.1. エディタ機能
- JSONテキストの貼り付け、直接編集
- JSONファイルのドラッグ＆ドロップ読み込み
- JSONファイルのアップロードボタンからの読み込み
- シンタックスエラーのリアルタイム検知とステータス表示
- JSONの整形（Format）と圧縮（Minify）
- クリップボードへのコピー
- エディタのクリア
- サンプルデータの読み込み

### 3.2. ツリー表示 (Tree View)機能
- JSONの階層構造をツリー形式で可視化
- ノードの展開・折りたたみ（個別および一括）
- キーおよび値によるインクリメンタル検索機能
- データ型に応じたカラーコーディング（文字列:緑、数値:青、真偽値:オレンジ、Null:赤）

### 3.3. 分析 (Analysis)機能
- **自動ターゲット検出**: ルートが配列でない場合でも、ネストされた最大のオブジェクト配列を自動的に見つけて分析対象とする。
- **データ概要**: 総レコード数、フィールド数、Null総数、Enum候補数、数値フィールド数、ネストフィールド数を表示。
- **Null率分析**: 各フィールドのデータ欠損率を計算し、バーチャートとパーセンテージで可視化。
- **Enum分析**: カーディナリティ（値の種類）が低いフィールドを自動検出し、値の出現回数をバーチャートで表示。
- **数値統計**: 数値フィールドの平均、最小、最大、合計を計算。
- **フィールド一覧**: 全フィールド（ネストされたパスも含む）の型、存在率、Null数、ユニーク値数、サンプル値のテーブル表示。

### 3.4. テーブル (Table)機能
- JSON配列データを表形式で表示。
- カラムごとのソート機能（昇順・降順トグル）。
- データ型に応じたテキストのカラーコーディング。

## 4. 受け入れテスト条件 (Acceptance Criteria)

### テストケース1: 基本的なJSONパースと可視化
- **条件**: 有効なJSON（オブジェクトまたは配列）をエディタに入力する。
- **期待結果**:
  - ステータスバーが「有効なJSON」となり緑色になること。
  - ツリー表示タブに正しく階層構造が描画されること。
  - 検索ボックスで特定のキーや値を検索し、ヒットしたノードがハイライトされること。

### テストケース2: エラーハンドリング
- **条件**: 構文エラーのある不正なJSONテキストを入力する。
- **期待結果**:
  - ステータスバーがエラー内容を示し、赤色になること。
  - ツリー、分析、テーブルの各ビューが「空の状態（Empty State）」の表示になること。

### テストケース3: 配列データの分析とテーブル表示
- **条件**: フラットな配列データ（例：サンプルデータ）を読み込む。
- **期待結果**:
  - 分析タブにて、総レコード数や各フィールドの統計（Null率、Enum等）が正しく計算され表示されること。
  - テーブルタブにて、全レコードが行として表示され、各列のヘッダーをクリックすると正しくソートされること。

### テストケース4: ネストされたJSONの自動検出と分析
- **条件**: ルートがオブジェクトであり、深い階層に配列を持つJSONを入力する。（例: `{"users": [{"id": 1, "profile": {"age": 30}}]}`）
- **期待結果**:
  - 分析タブおよびテーブルタブで、対象となる配列（例：`users`）が自動的に検出されること。
  - 「〇〇内の配列を分析対象として使用しています」というインフォメーションバナーが表示されること。
  - ネストされたフィールド（例: `profile.age`）が展開され、フィールド一覧に「└─ age (profile.age)」のようにインデント付きで表示されること。

### テストケース5: UI操作とレスポンシブデザイン
- **条件**: エディタパネルと可視化パネルの間にあるリサイザーをドラッグする。また、ウィンドウサイズをスマホサイズ（幅768px以下）に縮小する。
- **期待結果**:
  - リサイザーのドラッグによって左右のパネル幅が調整できること。
  - スマホサイズにした場合、左右の並びが上下の並びに変わり、適切にレイアウトされること。
